# Agent Coding Rules

1. **配置属性统一使用 `*Properties` 类**  
   - 所有需要注入的配置项必须定义在 `@ConfigurationProperties` 标注的 `XxxProperties` 类中，并通过构造器注入；禁止在业务类上使用裸 `@Value`。
2. **控制器入参超过 3 个时必须封装 DTO**  
   - 当接口需要 3 个以上的入参时，创建 `XxxRequest`/`XxxCommand` 类承载参数，避免长参数列表并便于校验。
3. **对外响应必须友好**  
   - 业务流程出现异常时，统一转换为领域异常并由上层返回结构化响应，严禁直接向调用方抛出原始异常堆栈。
4. **所有脚本/编码一律使用 UTF-8**  
   - 无论是临时 one-liner 还是持久脚本、源代码文件，都必须明确以 UTF-8（无 BOM）编码保存，避免 Windows/WSL 等环境之间出现乱码或解析失败。
5. **所有思考过程与交互内容统一使用中文**  
   - 在提交说明、代码评审、思考过程、终端输出描述等互动中，默认采用中文表达；若遇到外部规范要求其它语言，需在备注中说明原因。

> 说明：仓库曾混用 ANSI/GBK 与 UTF-8，任何新增或修改的文件都必须以 UTF-8 保存，并在提交前检查日志/注释是否存在乱码，发现后立刻修复。
