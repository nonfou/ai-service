# AI API Platform - Backend 文档

## 📖 项目简介

AI API Platform 是一个功能完整的 AI API 聚合平台后端系统，提供统一的 OpenAI 兼容接口，支持多个 AI 服务提供商的智能调度、计费管理、用户订阅等完整功能。

### 🎯 核心价值

- **多账户中继**: 智能管理多个 AI 服务商账户，实现负载均衡和故障转移
- **统一接口**: 提供 OpenAI 兼容的标准 API 接口
- **完整计费**: 支持余额管理、充值、订阅套餐、配额限制
- **高可用性**: 会话粘性、健康检查、自动降级
- **管理后台**: 完善的用户、订单、模型、账户管理功能

## 🚀 快速开始

### 环境要求

- Java 21+
- MySQL 8.0+
- Redis 6.0+
- Maven 3.8+

### 启动步骤

1. **配置数据库**
   ```bash
   # 创建数据库
   mysql -u root -p < src/main/resources/db/init_database.sql
   ```

2. **配置环境变量**
   ```bash
   cp .env.example .env
   # 编辑 .env 文���，配置数据库、Redis、邮件等
   ```

3. **启动应用**
   ```bash
   mvn spring-boot:run
   ```

4. **访问接口**
   - API 地址: http://localhost:8080
   - 健康检查: http://localhost:8080/actuator/health

详细部署文档请参考 [DEPLOYMENT.md](./DEPLOYMENT.md)

## 🔑 核心特性

### 1. 智能账户调度

支持多种调度策略：
- **轮询 (Round Robin)**: 均匀分配请求
- **最少使用 (Least Used)**: 优先使用使用次数最少的账户
- **优先级 (Priority)**: 按优先级选择账户
- **成本优化 (Cost Optimized)**: 选择成本最低的账户
- **混合策略 (Hybrid)**: 综合考虑多个因素

### 2. 会话粘性

同一用户的连续对话请求会路由到同一个后端账户，确保对话上下文的连续性。

### 3. 健康检查和故障转移

- 定期检查后端账户健康状态
- 自动降级故障账户
- 失败自动重试和切换

### 4. 完整计费系统

- 精确的 Token 计费（区分输入/输出/缓存 Token）
- 支持模型价格倍率
- 支持账户成本倍率
- 余额实时扣除
- 配额管理和告警

### 5. 多支付渠道

- 支付宝 PC 网站支付
- 微信 Native 扫码支付
- 订单自动对账
- 异步通知处理

### 6. 订阅套餐

- 灵活的套餐配置
- 自动额度发放
- 订阅历史记录
- 支持多种套餐类型

- **JWT 认证**: 无状态 Token 认证
