# 账户调度系统

## 📖 概述

账户调度系统是 AI API Platform 的核心组件之一，负责管理多个后端 AI 服务提供商账户，实现智能调度、负载均衡、会话粘性和故障转移等功能。

## 🎯 核心功能

- **多账户管理**: 支持 Copilot 和 OpenRouter 多个账户
- **智能调度**: 5种调度策略可选
- **会话粘性**: 同一会话路由到同一账户
- **健康检查**: 自动检测账户健康状态
- **故障转移**: 自动降级和切换故障账户
- **并发控制**: 限制单账户并发请求数
- **速率限制**: 限制单账户每分钟请求数
- **成本优化**: 支持成本倍率配置

## 🎲 调度策略

### 1. ROUND_ROBIN - 轮询

均匀分配请求到所有可用账户。

**算法逻辑**:
- 使用 AtomicInteger 实现线程安全的轮询
- 通过取模运算循环选择账户
- 每次调用计数器自增

**优点**:
- 简单公平
- 分布均匀

**缺点**:
- 不考虑账户负载
- 不考虑账户性能差异

---

### 2. LEAST_USED - 最少使用

优先选择使用次数最少的账户。

**算法逻辑**:
- 从 Redis 获取每个账户的使用次数
- 比较所有账户的使用次数
- 选择使用次数最少的账户
- 如果没有统计数据，默认使用次数为0

**优点**:
- 负载均衡
- 避免单点过载

**缺点**:
- 需要统计使用次数
- 可能频繁切换账户

---

### 3. PRIORITY - 优先级

按账户优先级选择，数字越小优先级越高。

**算法逻辑**:
- 比较所有账户的优先级字段
- 选择优先级值最小的账户
- 如果优先级相同，按列表顺序选择

**优点**:
- 可配置优先级
- 适合差异化账户管理

**缺点**:
- 高优先级账户可能过载
- 需要手动配置优先级

---

### 4. COST_OPTIMIZED - 成本优化

选择成本最低的账户。

**算法逻辑**:
- 计算每个账户的总成本: 账户成本倍率 × 模型价格倍率
- 获取模型配置信息
- 比较所有账户的综合成本
- 选择成本最低的账户

**优点**:
- 节省成本
- 适合价格敏感场景

**缺点**:
- 可能牺牲性能
- 低成本账户可能不稳定

---

### 5. HYBRID - 混合策略（推荐）

综合考虑优先级、使用率、错误率和成本等多个因素。

**算法逻辑**:
- 权重配置: 优先级30%、使用率30%、错误率20%、成本20%
- 计算每个账户的综合得分
- 优先级分数: 账户优先级值 / 100
- 使用率分数: 从使用统计获取
- 错误率分数: 从错误统计计算
- 成本分数: 账户成本与模型价格的综合
- 综合得分 = w1×优先级 + w2×使用率 + w3×错误率 + w4×成本
- 选择得分最低的账户

**优点**:
- 综合优化
- 适应性强
- 可调整权重

**缺点**:
- 计算复杂
- 需要调优权重

## 🔗 会话粘性

### 概念

同一用户的连续对话请求会路由到同一个后端账户，确保对话上下文的连续性。

### 实现机制

**核心方法**:
- **generateSessionHash()**: 生成会话哈希
  - 输入: 用户ID + 对话ID
  - 算法: SHA-256 哈希
  - 输出: 唯一会话标识

- **createSessionMapping()**: 创建会话映射
  - 记录会话哈希、后端账户ID、API密钥ID、用户ID
  - 初始化请求计数为1
  - 设置过期时间(默认1小时)
  - 保存到数据库

- **getSessionMapping()**: 查询会话映射
  - 根据会话哈希查询映射记录
  - 返回关联的账户信息

- **incrementRequestCount()**: 更新请求计数
  - 请求计数加1
  - 延长过期时间(再延长1小时)

### 完整流程

```
1. 提取会话标识
   ├─ 用户ID
   ├─ 对话ID (从消息历史推断)
   └─ 生成会话哈希 SHA-256(userId:conversationId)

2. 查询会话映射
   ├─ 存在且未过期
   │   ├─ 检查账户健康状态
   │   ├─ 健康 → 使用已绑定账户
   │   └─ 不健康 → 重新调度
   └─ 不存在或已过期 → 重新调度

3. 重新调度
   ├─ 执行调度策略选择账户
   ├─ 创建新的会话映射
   └─ 设置过期时间（默认 1 小时）

4. 使用账户进行 API 调用

5. 更新会话映射
   ├─ 请求计数 +1
   └─ 延长过期时间
```

## 🏥 健康检查

### 自动健康检查

**执行机制**:
- 定时任务: 每30秒执行一次
- 检查范围: 所有活跃状态的账户
- 检查方法: 发送测试请求到后端API

**检查流程**:
1. 获取所有活跃账户列表
2. 遍历每个账户执行健康检查
3. 创建简单测试请求
4. 获取对应的代理服务
5. 发送测试请求并等待响应
6. 判断响应是否成功

**不健康处理**:
1. 增加账户错误计数
2. 记录最后错误时间和错误信息
3. 如果错误次数达到阈值(默认3次)
4. 将账户状态降级为 ERROR
5. 发送告警通知管理员
6. 更新数据库记录

**账户恢复**:
1. 继续对降级账户执行健康检查
2. 如果检查通过
3. 将状态恢复为 ACTIVE
4. 重置错误计数和错误信息
5. 记录恢复日志

## 🚦 并发和速率控制

### 并发控制

**并发限制检查**:
- Redis键格式: `account_concurrent:{accountId}`
- 获取当前并发数
- 与账户配置的最大并发数比较
- 达到限制返回false,未达到返回true

**增加并发计数**:
- 使用Redis INCR命令原子增加
- 设置过期时间10分钟
- 防止计数器永久存在

**减少并发计数**:
- 使用Redis DECR命令原子减少
- 请求完成后调用

### 速率限制

**速率限制检查(令牌桶算法)**:
- Redis键格式: `account_rate:{accountId}:{yyyyMMddHHmm}`
- 按分钟粒度统计
- 获取当前分钟的请求计数
- 与配置的每分钟限制比较
- 达到限制返回false

**增加速率计数**:
- 生成当前分钟的键
- 使用Redis INCR增加计数
- 设置过期时间2分钟
- 自动清理过期数据

## 📊 账户选择完整流程

**AccountSchedulerService.selectAccount() 方法**:

1. **检查会话粘性**
   - 如果提供了sessionHash
   - 查询会话映射记录
   - 检查映射是否存在且未过期
   - 获取绑定的后端账户
   - 检查账户健康状态
   - 如果账户健康,使用该账户并更新请求计数
   - 如果不健康,继续下一步

2. **获取可用账户列表**
   - 根据模型查询支持该模型的账户
   - 如果列表为空,抛出"没有可用的后端账户"异常

3. **过滤账户**
   - 检查账户状态(必须为ACTIVE)
   - 检查并发限制(未达到最大并发)
   - 检查速率限制(未达到每分钟限制)
   - 检查错误率(错误率不超过50%)
   - 过滤后如果列表为空,抛出"所有账户都达到限制"异常

4. **根据策略选择账户**
   - 获取配置的调度策略
   - 创建调度上下文(包含模型和API密钥)
   - 调用对应策略的选择方法
   - 返回选中的账户

5. **创建会话映射**
   - 如果提供了sessionHash
   - 创建新的会话映射记录
   - 绑定账户ID、API密钥ID、用户ID
   - 设置过期时间

6. **更新使用统计**
   - 增加账户使用计数
   - 用于统计和后续调度决策

**filterAccounts() 过滤逻辑**:
- 使用Stream API进行链式过滤
- 检查账户状态是否为ACTIVE
- 调用并发控制服务检查并发限制
- 调用速率限制服务检查速率限制
- 计算账户错误率,拒绝错误率过高的账户

## 🔧 配置

### application.yml 配置说明

**调度策略配置**:
- 策略选项: ROUND_ROBIN, LEAST_USED, PRIORITY, COST_OPTIMIZED, HYBRID
- 默认推荐: HYBRID

**会话粘性配置**:
- enabled: 是否启用会话粘性
- ttl-hours: 会话映射有效期(小时)

**健康检查配置**:
- enabled: 是否启用自动健康检查
- interval-seconds: 检查间隔秒数
- failure-threshold: 失败阈值,达到后降级账户

**并发控制配置**:
- default-max-concurrent: 默认最大并发数

**速率限制配置**:
- default-rate-limit: 默认每分钟请求数限制

**混合策略权重配置**:
- priority: 优先级权重(0-1)
- usage: 使用率权重(0-1)
- error: 错误率权重(0-1)
- cost: 成本权重(0-1)

## 📊 监控指标

### Redis 统计键

**并发计数**:
- 键: `account_concurrent:{accountId}`
- 类型: Integer
- 说明: 当前账户的并发请求数

**速率计数**:
- 键: `account_rate:{accountId}:{yyyyMMddHHmm}`
- 类型: Integer
- 说明: 指定分钟内的请求次数

**使用统计**:
- 键: `account_usage:{accountId}`
- 类型: Integer
- 说明: 账户累计使用次数

**错误统计**:
- 键: `account_errors:{accountId}:{yyyyMMddHH}`
- 类型: Integer
- 说明: 指定小时内的错误次数

**账户健康状态**:
- 键: `account_health:{accountId}`
- 类型: JSON
- 说明: 账户健康检查结果

### 监控查询

**AccountMetrics 指标对象**包含:
- accountId: 账户ID
- currentConcurrent: 当前并发数
- currentRate: 当前速率(每分钟请求数)
- totalUsage: 累计使用次数
- errorRate: 错误率
- lastUsedAt: 最后使用时间

## 🔍 常见问题

### Q: 如何选择调度策略？

A:
- **轮询**: 简单场景，账户性能相近
- **最少使用**: 需要均衡负载
- **优先级**: 有明确的账户优先级
- **成本优化**: 成本敏感场景
- **混合**: 生产环境推荐，综合优化

### Q: 会话粘性有什么限制？

A:
- 默认有效期 1 小时
- 账户不健康时会自动切换
- 需要客户端传递对话 ID

### Q: 如何添加新的后端账户？

A: 通过管理后台或 API 添加，需要提供：
- 账户名称
- 提供商类型（copilot/openrouter）
- Access Token（会自动加密）
- 优先级、并发限制、速率限制等配置

### Q: 账户降级后如何恢复？

A: 健康检查会自动尝试恢复，连续成功后状态自动变为 active。也可以手动在管理后台启用。
